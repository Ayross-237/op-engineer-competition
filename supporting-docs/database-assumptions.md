# Below lists all the restrictive assumptions made when designing the database schema (that enforce some type of rule)
- Students can enrol in a program at most once (they cannot be listed more than once on the same weekly slot, e.g. JPC Tuesday 4:30-7:30pm)
- Caterers cater for an entire school, which means all programs hosted at that school have the same caterer
- Programs only run on weekdays (Monday–Friday); the `day_of_week` enum does not include Saturday or Sunday. Sessions inherit this transitively since each session belongs to a program.
- Year levels are integers only (e.g. 9, 10, 11, 12); the schema cannot represent letter-based years such as Prep or Kindy so it does not support primary schools that use them
- Each caterer has exactly one main point of contact (`contact_email`); the schema cannot represent multiple equally-weighted contacts on the caterer side
- `chef_email` is nullable on the assumption that when the chef is the main point of contact, only `contact_email` is populated rather than duplicating the address into both columns
- A program has at most one session per date; the composite primary key `(program_id, date)` on `sessions` prevents two sessions of the same program on the same day
- The schema does not enforce that an `absences` row references a program the student is actually enrolled in — application code is responsible for filtering absences against enrolments when reporting
- `sub_manager_*` columns are independently nullable; the schema does not require them to be all-or-nothing populated together (e.g. a row could in principle have `sub_manager_name` set without a mobile), so consumers must treat any `sub_manager_*` value as a signal that a substitute is running the session
- `dietary_extra` is unstructured free text for restrictions that don't fit the `dietary_tag` enum (e.g. specific allergies). The schema does not validate or normalise its contents, so consumers must treat it as a human-readable hint rather than a machine-actionable filter — automated dish selection should still rely on the structured `dietary` array
